# startup

Небольшой лендинг студии/команды, которая делает веб-приложения полного цикла: от discovery и архитектуры до разработки, production-деплоя и поддержки. Проект выполнен как статическая одностраничная витрина без сборщика и backend-части.

## Что есть на странице

- hero-блок с анимированным canvas-фоном, scroll progress и крупным позиционированием студии;
- блок услуг с интерактивными табами: `Discovery`, `Architecture`, `Build`, `Deploy`, `AI`, `Support`;
- секция процесса разработки из 7 этапов с анимируемой timeline;
- портфолио-блок с live-кейсом `AiNoCraft` и карточкой под следующий проект;
- секция технологического стека;
- блок команды с двумя ролями;
- контактная форма, которая не отправляет данные на сервер, а собирает `mailto:`-черновик письма.

## Особенности реализации

Страница собрана на чистом HTML, CSS и JavaScript.

Основные интерактивные элементы:

- переключение языка `RU / EN` через встроенный `i18n`-словарь в `script.js`;
- переключение темы `dark / light` с сохранением выбора в `localStorage`;
- анимированный `flow-field` canvas на фоне;
- reveal-анимации блоков при скролле;
- tilt-эффект карточек при наведении;
- анимация строк кода в hero-блоке;
- интерактивный переключатель сервисов;
- обновляемый progress-indicator для секции процесса;
- форма контакта через `mailto:hello@startup.dev`.

В коде также учтён `prefers-reduced-motion`: часть эффектов отключается, если у пользователя сниженная анимация в системе.

## Структура проекта

```text
startup/
├── index.html
├── style.css
└── script.js
```

## Файлы проекта

- `index.html` — вся разметка лендинга и секций;
- `style.css` — визуальный стиль, темы, layout, анимации и адаптивность;
- `script.js` — i18n, theme toggle, scroll UI, canvas-анимация, tabs, timeline и логика контактной формы.

## Технологии

- HTML5
- CSS3
- Vanilla JavaScript
- Google Fonts (`IBM Plex Sans`, `IBM Plex Mono`, `Sora`)

## Как запустить

Так как это статическая страница, достаточно открыть `index.html` в браузере.

Если нужен локальный сервер для удобства, можно использовать любой простой вариант, например VS Code Live Server.

## Что важно знать

- у проекта нет сборки и package manager;
- контактная форма не отправляет данные в backend, а открывает почтовый клиент через `mailto:`;
- часть внешних ссылок в шаблоне пока оставлена как заглушка, например Telegram в шапке и блоке контактов;
- проект ориентирован на презентацию услуг и кейсов, а не на полноценное web-приложение.

## Итог

`startup` — это аккуратный статический лендинг с сильным визуальным слоем и хорошим количеством клиентских интеракций: двуязычность, темы, canvas-фон, tabs, timeline и mailto-формой. Для следующего шага сюда логично добавлять реальные кейсы, рабочие контактные ссылки и при необходимости подключать backend-обработку формы.
